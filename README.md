# HTTP客户端工具 - 增强版 v2.0

一个功能强大的类似Postman的HTTP/HTTPS客户端工具，支持多标签页和历史记录管理。

## 🎉 主要特性

### ✅ 完整功能
- **多标签页支持** - 同时测试多个API请求
- **导入 curl** - 粘贴 curl 命令，解析为完整请求并在新标签页展示
- **文件上传** - 支持 multipart/form-data 上传单个或多个文件
- **历史记录管理** - 自动保存请求历史
- **分类管理** - 按项目/模块组织请求
- **实时搜索** - 快速查找历史请求
- **数据导入导出** - 备份和分享请求配置
- **快捷键支持** - 提升操作效率

### 🔧 支持的HTTP方法
- GET、POST、PUT、DELETE
- PATCH、HEAD、OPTIONS

## 🚀 快速开始

### 创建新标签页
- 点击工具栏"新建请求"按钮
- 或使用快捷键 `Ctrl+N`

### 导入 curl 命令
1. 点击工具栏"导入 curl"或菜单 文件 → 导入 curl
2. 在弹窗中粘贴完整的 curl 命令（支持 `-X`、`-H`、`-d`、`-u`、`-b`、`--url` 等常用选项）
3. 点击确定后，解析结果会在**新标签页**中展示（方法、URL、Headers、Body 均会填充）
4. 可直接在该标签页编辑或发送请求

### 发送请求
1. 在标签页中输入URL
2. 选择HTTP方法
3. 设置Headers和Body（如需要）
4. 点击"发送"按钮或按 `Ctrl+Enter`

### 上传文件
1. 将方法切换为 `POST`、`PUT`、`PATCH` 或 `DELETE`
2. 打开请求区的 `Files` 标签页
3. 点击"添加文件"选择一个或多个文件，并输入上传字段名（默认 `file`）
4. 如需额外表单字段，可在 `Body` 中填写 JSON 对象；非 JSON 内容会作为 `body` 字段发送
5. 点击"发送"，程序会自动使用 `multipart/form-data` 并生成正确的边界

### 保存请求到历史
1. 点击"保存"按钮或按 `Ctrl+S`
2. 输入请求名称
3. 选择分类
4. 点击确定

### 加载历史记录
1. 在右侧历史面板中找到请求
2. 双击记录即可加载到当前标签页

### 关闭标签页
- 点击标签页右上角的 `×` 按钮
- 至少保留一个标签页

## 🎯 使用技巧

### 1. 多标签页工作流
```
场景: 并行测试多个API端点

步骤:
1. 点击"新建请求"创建标签页1
2. 输入GET请求 → 发送
3. 再次点击"新建请求"创建标签页2
4. 输入POST请求 → 发送
5. 标签页3: PUT请求
6. 标签页4: DELETE请求
7. 同时查看多个请求结果
```

### 2. 历史记录管理
```
场景: 组织API测试请求

步骤:
1. 为不同项目创建分类:
   - 项目A (API v1)
   - 项目B (API v2)
   - 开发测试

2. 保存请求时选择相应分类

3. 通过分类快速筛选:
   菜单 → 选择分类 → 查看所有相关请求

4. 搜索功能:
   在搜索框输入"get" → 快速找到所有GET请求
```

### 3. 导入 curl 命令
```
场景: 从浏览器/文档复制 curl 命令快速测试

步骤:
1. 菜单栏 → 文件 → 导入 curl（或工具栏「导入 curl」）
2. 粘贴完整 curl 命令（如 curl -X POST 'https://api.example.com' -H 'Content-Type: application/json' -d '{}'）
3. 确定后在新标签页中自动填充方法、URL、Headers、Body
4. 可编辑后直接发送或保存到历史
```

### 4. 导入导出请求
```
导出:
1. 菜单栏 → 文件 → 导出请求
2. 选择保存位置
3. 生成JSON文件

导入:
1. 菜单栏 → 文件 → 导入请求
2. 选择JSON文件
3. 自动加载到历史记录
```

## 🔧 故障排除

### 问题1: 标签页关闭按钮不显示
**解决方案**: 确保使用 `http_client_enhanced.py` 版本

### 问题2: 历史记录不保存
**检查**:
- 数据库文件权限
- 程序是否正常退出

### 问题3: 导入失败
**解决方案**:
- 检查JSON文件格式
- 确保文件编码为UTF-8
